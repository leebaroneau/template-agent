# Onboarding

Complete these before any other work. Capture answers into the right memory file as you go. Delete this file when every question has an answer recorded.

## Foundation → write to `memories/USER.md`

1. **Who is the board / primary user?** Name, role, primary contact email.
2. **What is the user's primary working surface?** (e.g. ecommerce storefront, API services, content platform, financial advisory, partnership operations)
3. **What hard rules are non-negotiable?** Deployment rules, voice rules, off-limits actions.
4. **Brand or organization context.** What the user is building, who they serve, the rough scale.

## Insights → write to `memories/USER-INSIGHTS.md`

1. **How does the user prefer to receive output?** Concise vs detailed, bullets vs prose, recommendation-first vs options.
2. **What's the user's working tempo?** Daily syncs, async, urgent escalations only.
3. **What patterns of past mistakes should you remember to avoid?**
4. **What does the user reliably appreciate when you do it?**

## Role-specific (optional)

If your role has domain context the user wants documented, ask one or two follow-ups appropriate to your title. Two questions max.
